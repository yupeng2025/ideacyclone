# 贡献指南 (Contributing to IdeaCyclone)

首先，感谢您抽出时间参与贡献！❤️

我们需要并重视所有类型的贡献。请在提交贡献之前阅读相关部分。这将使我们维护者的工作更轻松，也会让您的贡献体验更顺畅。社区期待您的参与。

## 我有问题

如果您有问题，我们建议您先阅读现有的 [文档](README.md)。

在提问之前，最好先搜索已有的 [Issues](https://github.com/YOUR_USERNAME/ideacyclone/issues)，看看是否能解决您的问题。如果您找到了相关 Issue 但仍需澄清，可以在该 Issue 下留言。与直接创建新 Issue 相比，这通常能更快得到回应。

## 我想贡献

### 报告 Bug

-   确保您使用的是最新版本。
-   阅读 [文档](README.md) 确认该行为是否符合预期。
-   搜索 Issue 列表，查看该问题是否已被报告。如果是，请在现有 Issue 下补充信息，而不是新建。

### 建议新功能

-   搜索 Issue 列表，查看该功能建议是否已被提出。如果是，请在现有 Issue 下参与讨论。

### 第一次提交代码

1.  Fork 本项目。
2.  为您的功能或修复创建一个新分支 (`git checkout -b feature/AmazingFeature`)。
3.  提交您的更改 (`git commit -m 'Add some AmazingFeature'`)。
4.  推送到您的 Fork (`git push origin feature/AmazingFeature`)。
5.  提交 Pull Request。

## 规范指南

### 提交信息 (Commit Messages)

-   使用祈使句（例如 "Add feature" 而不是 "Added feature"）。
-   第一行限制在 72 个字符以内。
-   如果相关，请在第一行之后引用 Issue 或 Pull Request 编号。
